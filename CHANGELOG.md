# 更新日志

本文件用于记录项目每次功能和行为变化。

## 统计

- 累计更新次数：5

## 记录规则

- 每次提交前必须新增一条记录。
- 记录格式为“日期 + 当日第 N 次更新（更新 #编号）”。
- 先写用户可感知变化，再写内部重构。
- 必须包含影响文件列表。

## 2026-04-16

### 第 1 次更新（更新 #001）

#### 新增

- 新增 Alt + Right 重置功能：测量过程中可随时重置并回到第一步。
- 主界面控制说明新增重置快捷键提示。

- 测量流程改为可重启循环，重置后不需要退出程序重开。

#### 影响文件

- README.md
- CHANGELOG.md

### 第 4 次更新（更新 #004）

#### 新增

- 主窗口新增菜单栏：Settings -> Language，可在 English/Chinese 间切换。
- 软件默认英文显示；切换中文后，主界面和测量悬浮提示同步切换为中文。

#### 优化

- 多语言文案统一由 i18n 字典按 key 管理，减少散落硬编码文本。
- 抽象测量流程中的重复提示与双点采集逻辑，降低重复代码成本。
- 预解析 Alt 修饰键并复用，减少采点热路径中的重复字符串拆分开销。
- 悬浮提示窗口改为 Toplevel，避免额外创建 Tk 根窗口。

#### 影响文件

- main.py
- CHANGELOG.md

### 第 5 次更新（更新 #005）

#### 新增

- 新增主窗口“首页/设置页”单窗口切换，设置项不再直接挂在菜单下操作。
- 新增 Alt+Q 触发窗口设置项（0.3/0.5/0.8 秒），并支持设置持久化。
- 新增开发者流程说明文档，明确启动、退出、防误触与重置状态机逻辑。

#### 修复

- 修复流程中 Alt+Q 退出体感滞后问题：在关键等待段即时消费退出请求并优先隐藏左上角提示。
- 修复流程期间主程序偶发“未响应”问题：等待循环内持续泵主窗口事件。

#### 优化

- 优化热键逻辑：Alt+Q 启动与流程中退出共用按下时序判断，降低语音 Alt 键误触概率。
- 优化流程控制：区分 reset/exit/done 状态，流程行为更可预测。
- 优化本地开发体验：补充 .venv/.conda/settings.json 忽略规则，避免无关文件进入版本库。

#### 影响文件

- main.py
- README.md
- .gitignore
- requirements.txt
- CHANGELOG.md
